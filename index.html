1 模块 module

发表于 2024-04-22 更新于 2024-04-23

1 基本

  1. node中引入模块，三个步骤：
    1. 路径分析
    2. 文件定位
    3. 编译执行
  2. node中模块分为三类：
    1. node提供的模块，核心模块（编译进二进制文件，只有路径分析这一步骤，加载速度快）
    2. 用户编写的模块，文件模块（运行时动态加载，加载速度慢）
    3. 第三方模块（自定义模块）

2 模块使用

2.1 module.exports

3 缓存

node对require引入的模块都会进行缓存，缓存的是编译和执行后的对象。当二次加载时，会先检查缓存。核心模块缓存检查优先级高于文件模块
所有编译成功的模块都会将其文件路径作为索引缓存在Module._cache对象上

4 路径分析和文件定位

  1. require()接受一个标识符作为参数，针对标识符进行模块查找
  2. 标识符分类：
    1. 核心模块，如http,fs,path等
    2. 相对路径文件
    3. 绝对路径文件
    4. 非路径形式的文件模块（require(express)等，引入用户自定义模块）

4.1 自定义模块

加载速度慢
nodejs将每个文件都视为一个module
require(module)之后，就会去立即执行对应模组

4.1.1 模块路径

node定位文件模块的查找策略：（和原型链类似）先找当前目录底下的，没有则去找父目录底下的，以此类推直到根目录

5 模块编译

定位文件后，建立模块对象，然后载入编译（不同扩展名载入方式不同）
主要有：

  1. .js文件：通过fs模块同步读取文件后执行
  2. .node文件：用C/C++编写的拓展文件。通过dlopen()方法加载后编译生成的文件
  3. .json文件：通过fs模块同步读取后，用JSON.parse()解析返回结果
  4. 其余拓展文件：都被当作.js读取

5.1 js模块的编译

Node会对获取的js文件用function wrapper包裹，从而达到作用域隔离的效果
包裹的时候会为每个js文件添加五个参数：

  1. exports
  2. require
  3. module
  4. __filename
  5. __dirname 包裹之后用vm原生模块的runInThisContext()方法执行，返回一个function对象，然后将五个参数传入执行该函数

5.1.1 exports和module.exports区别

原因：从上面看exports和module都是作为参数传入的，而js的参数都是值传递。
如果直接改变exports的赋值，相当于改变stack中的值，对heap中的值不会有影响，也不会影响作用域外的值
而改变module.exports相当于改变heap当中的值

5.2 C/C++模块

Node调用process.dlopen()方法加载执行
.node文件是C/C++模块编译后生成的，因此只需要加载和执行即可

5.3 JSON文件

fs模块读取+JSON.parse()方法得到对象即可

计算机内核对于IO的处理：阻塞和非阻塞

  1. 阻塞：CPU等待IO处理
  2. 非阻塞：调用之后立即返回调用状态，CPU去处理其他事务。因此，需要有方法检测IO是否完成，叫做“轮询”
    轮询技术主要有：
  3. read：重复调用检测IO是否完成
  4. select：对文件描述符上的事件状态进行判断
  5. poll：
  6. epoll：
  7. kqueue：
    轮询技术仍然是一种同步技术，因为程序仍然需要等待IO完全返回。最好的情况是，IO立即返回之后，无需轮询先去处理其他操作，当IO处理完毕之后通过信号或者回调将数据返回。
    多线程处理的方式：一个线程处理等待IO和轮询，另一个线程处理其他操作（但是node是单线程）

1 Node的异步IO

1.1 事件循环

  1. node的执行模型
  2. 进行启动，node会类似于while循环，每一次循环体成为Tick。每一个Tick过程，就是检查当前是否有时间需要处理，有则处理相关事件；没有，则进入下一个循环
    ![Pasted image 20240419155259.png]

1.2 观察者

  1. 用于判断当前哪些事件需要执行
  2. Node中，每次出现一个事件都有对应观察者，观察者会对事件进行分类。然后Node询问观察者，来处理事件

1.3 请求对象

异步IO的回调函数有os执行，从js发起调用到内核执行IO操作，中间存在一个请求对象
一层一层调用，将js层传入的参数和方法都分装到请求对象中，并且将请求对象推入线程池中运行（然后进行等待，如果线程池有可用线程，就会执行相应方法）。请求对象推入线程池后，js调用就可以立即返回，执行其他操作，从而达到异步的效果。

1.4 执行回调

线程池中的IO操作执行完之后，将结果存储，然后通知IOCP当前操作完成。同时，每次Tick执行中，它会调用IOCP相关方法检查线程池中是否有执行完的请求。如果有，将请求对象加入到IO观察者的队列中，之后作为事件处理
